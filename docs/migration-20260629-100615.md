# Migration Guide: `final_amount` Field Added to `OrderOut` Schema

**Date:** 2026-06-29
**Severity:** CRITICAL — affects every order endpoint, the frontend, seed pipelines, and the full test suite.

---

## Summary

A new non-nullable integer field, `final_amount`, has been added to the `OrderOut` response schema. Because this field is **required and non-nullable**, every layer of the application that constructs, serializes, seeds, or tests an `Order` object must be updated before deployment. Failure to do so will cause runtime `ValidationError` exceptions on all order endpoints (resulting in HTTP 500 responses), broken database seed pipelines, a failing test suite, and missing order total information in the frontend UI. This guide covers every required change across backend, frontend, and QA roles.

---

## Breaking Changes

### 1. `OrderOut` Pydantic Schema — New Required Field

The `OrderOut` schema now includes a non-nullable `final_amount: int` field. Any code path that instantiates or returns `OrderOut` without this field will raise a `pydantic.ValidationError` at runtime.

**Before:**
```python
class OrderOut(BaseModel):
    id: int
    user_id: int
    status: str
    created_at: datetime
```

**After:**
```python
class OrderOut(BaseModel):
    id: int
    user_id: int
    status: str
    created_at: datetime
    final_amount: int  # NEW — required, non-nullable
```

---

### 2. SQLAlchemy ORM Model — Missing Database Column

The `Order` ORM model does not yet have a `final_amount` column. Without it, queries will raise `AttributeError` or Pydantic will fail to serialize the response.

**Before:**
```python
class Order(Base):
    __tablename__ = "orders"

    id = Column(Integer, primary_key=True)
    user_id = Column(Integer, ForeignKey("users.id"), nullable=False)
    status = Column(String, nullable=False)
    created_at = Column(DateTime, default=datetime.utcnow)
```

**After:**
```python
class Order(Base):
    __tablename__ = "orders"

    id = Column(Integer, primary_key=True)
    user_id = Column(Integer, ForeignKey("users.id"), nullable=False)
    status = Column(String, nullable=False)
    created_at = Column(DateTime, default=datetime.utcnow)
    final_amount = Column(Integer, nullable=False, default=0)  # NEW
```

> ⚠️ A database migration script (e.g., Alembic) must be generated and applied. See [Required Actions](#required-actions-by-role).

### 3. `order_service.py` — `OrderOut` Construction Lacks `final_amount`

The service layer returns `OrderOut` instances without computing `final_amount`, causing a `ValidationError` on every order response.

**Before:**
```python
def get_order(order_id: int, db: Session) -> OrderOut:
    order = db.query(Order).filter(Order.id == order_id).first()
    if not order:
        raise HTTPException(status_code=404, detail="Order not found")
    return OrderOut.from_orm(order)
```

**After:**
```python
def get_order(order_id: int, db: Session) -> OrderOut:
    order = db.query(Order).filter(Order.id == order_id).first()
    if not order:
        raise HTTPException(status_code=404, detail="Order not found")
    # Ensure final_amount is computed before serialization
    if order.final_amount is None:
        order.final_amount = _compute_final_amount(order)
    return OrderOut.from_orm(order)

def _compute_final_amount(order: Order) -> int:
    """
    Compute and return the final order total in the smallest currency unit (e.g., cents).
    Cast explicitly to int — do NOT pass a float or Decimal directly into this field.
    """
    raw_total = sum(item.unit_price * item.quantity for item in order.items)
    discount = order.discount_amount or 0
    return int(round(raw_total - discount))
```

> ⚠️ **Precision Warning:** `final_amount` is typed as `int`. Always use `int(round(...))` when computing from `float` or `Decimal` values to avoid silent precision loss or a `TypeError`. If sub-cent precision is required, consider changing the field type to `Decimal`/`float` before merging.

### 4. `seed.py` — Seed Records Missing `final_amount`

Seeding without `final_amount` will raise an `IntegrityError` because the column is `NOT NULL` at the database level.

**Before:**
```python
orders = [
    Order(user_id=1, status="pending"),
    Order(user_id=2, status="completed"),
]
```

**After:**
```python
orders = [
    Order(user_id=1, status="pending",   final_amount=0),
    Order(user_id=2, status="completed", final_amount=4999),
]
```

### Updated `order_service.py` (critical path)

```python
from sqlalchemy.orm import Session
from fastapi import HTTPException
from backend.app.models.order import Order, OrderOut


def _compute_final_amount(order: Order) -> int:
    """
    Derive the final order amount from line items and discounts.
    Returns an int (e.g., total in cents). Always rounds before casting.
    """
    subtotal = sum(
        item.unit_price * item.quantity
        for item in (order.items or [])
    )
    discount = order.discount_amount or 0
    return int(round(subtotal - discount))


def get_order(order_id: int, db: Session) -> OrderOut:
    order = db.query(Order).filter(Order.id == order_id).first()
    if not order:
        raise HTTPException(status_code=404, detail="Order not found")

    # Guard: compute if not already stored
    if order.final_amount is None:
        order.final_amount = _compute_final_amount(order)

    return OrderOut.from_orm(order)


def list_orders(db: Session) -> list[OrderOut]:
    orders = db.query(Order).all()
    for order in orders:
        if order.final_amount is None:
            order.final_amount = _compute_final_amount(order)
    return [OrderOut.from_orm(o) for o in orders]
```
