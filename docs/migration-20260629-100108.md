# Migration Guide: `calculate_discount()` Signature Change & VIP Discount Rate Update

**Date:** 2026-06-29
**Severity:** CRITICAL / HIGH
**Affected Files:** 3

---

## Summary

Two breaking changes were introduced to the discount calculation system. First, the `coupon_code` parameter in `calculate_discount()` was moved from a positional argument to a **keyword-only argument**, meaning any call that passes `coupon_code` as the third positional value will raise a `TypeError` at runtime. Second, the VIP discount rate (`GRADE_DISCOUNT_RATE[UserGrade.VIP]`) was increased from **10% to 15%**, which will silently invalidate any test assertions or business logic that assumed the old rate. Both changes affect live order processing paths (`order_service.py`, `orders.py`) and the test suite (`test_discount.py`), making immediate action required across Backend and QA roles.

---

## Breaking Changes

### 1. `calculate_discount()` — `coupon_code` Moved to Keyword-Only

The `coupon_code` parameter is no longer accepted as a positional argument. It **must** now be passed by name.

| | Before | After |
|---|---|---|
| **Parameter kind** | `positional` | `keyword-only` |
| **Positional call** | ✅ Accepted | ❌ `TypeError` |
| **Keyword call** | ✅ Accepted | ✅ Accepted |

**Before:**
```python
# Function signature
def calculate_discount(subtotal: int, grade: UserGrade, coupon_code: str | None = None) -> int:
    ...

# Valid call (positional)
calculate_discount(500, UserGrade.VIP, "SUMMER20")
```

**After:**
```python
# Function signature
def calculate_discount(subtotal: int, grade: UserGrade, *, coupon_code: str | None = None) -> int:
    #                                                    ^ keyword-only boundary
    ...

# Only valid call form
calculate_discount(500, UserGrade.VIP, coupon_code="SUMMER20")

# This will now FAIL at runtime:
calculate_discount(500, UserGrade.VIP, "SUMMER20")
# TypeError: calculate_discount() takes 2 positional arguments but 3 were given
```

---

### 2. `GRADE_DISCOUNT_RATE[UserGrade.VIP]` — Rate Increased from 10% to 15%

| | Before | After |
|---|---|---|
| **Discount rate** | `0.10` (10%) | `0.15` (15%) |
| **Discount on $500 order** | `$50` | `$75` |
| **Final price on $500 order** | `$450` | `$425` |

**Before:**
```python
GRADE_DISCOUNT_RATE = {
    UserGrade.VIP: 0.1,   # 10%
    # ...
}
```

**After:**
```python
GRADE_DISCOUNT_RATE = {
    UserGrade.VIP: 0.15,  # 15%
    # ...
}
```

> ⚠️ **Business Impact:** This is not just a test issue. Orders near profit margin thresholds for VIP customers may now be unprofitable. Coordinate with stakeholders before deploying to production.

## Code Examples

### Fix 1: Update `calculate_discount()` Calls (Keyword-Only `coupon_code`)

**`backend/app/services/order_service.py` — Before:**
```python
def apply_order_discount(order: Order, user: User) -> int:
    return calculate_discount(order.subtotal, user.grade, order.coupon_code)
    #                                                     ^^^^^^^^^^^^^^^^^ positional — BROKEN
```

**After:**
```python
def apply_order_discount(order: Order, user: User) -> int:
    return calculate_discount(order.subtotal, user.grade, coupon_code=order.coupon_code)
    #                                                     ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^ keyword-only — FIXED
```

---

**`backend/tests/test_discount.py` — Before:**
```python
def test_vip_with_coupon():
    result = calculate_discount(500, UserGrade.VIP, "SUMMER20")  # BROKEN: positional coupon_code
    assert result == 50  # BROKEN: assumes 10% rate

def test_standard_with_coupon():
    result = calculate_discount(200, UserGrade.STANDARD, "SAVE10")  # BROKEN: positional coupon_code
    assert result == 20
```

**After:**
```python
def test_vip_with_coupon():
    result = calculate_discount(500, UserGrade.VIP, coupon_code="SUMMER20")  # FIXED
    assert result == 75  # FIXED: reflects 15% VIP rate

def test_standard_with_coupon():
    result = calculate_discount(200, UserGrade.STANDARD, coupon_code="SAVE10")  # FIXED
    assert result == 20  # Unchanged: STANDARD rate not affected

def test_coupon_code_positional_raises():
    """Regression: coupon_code must be keyword-only."""
    with pytest.raises(TypeError):
        calculate_discount(500, UserGrade.VIP, "SUMMER20")  # Must raise TypeError
```

### Fix 2: Update VIP Discount Assertions

**Before:**
```python
# Any hardcoded VIP discount expectations
VIP_DISCOUNT_ON_500  = 50   # 500 * 0.10
VIP_FINAL_PRICE_500  = 450  # 500 - 50

assert calculate_discount(500, UserGrade.VIP) == 50
assert get_final_price(order) == 450
```

**After:**
```python
# Updated to reflect 15% VIP rate
VIP_DISCOUNT_ON_500  = 75   # 500 * 0.15
VIP_FINAL_PRICE_500  = 425  # 500 - 75

assert calculate_discount(500, UserGrade.VIP) == 75
assert get_final_price(order) == 425
```

### Fix 3: Margin/Profit Threshold Guards in Service Layer

If `order_service.py` contains hardcoded profit margin logic tied to the old VIP rate:

**Before:**
```python
VIP_RATE = 0.10  # local copy or assumption
if (order.subtotal * (1 - VIP_RATE)) < MIN_PROFITABLE_ORDER:
    raise OrderNotProfitableError(...)
```

**After:**
```python
# Reference the source-of-truth constant directly rather than duplicating it
from app.discounts import GRADE_DISCOUNT_RATE

rate = GRADE_DISCOUNT_RATE.get(user.grade, 0)
if (order.subtotal * (1 - rate)) < MIN_PROFITABLE_ORDER:
    raise OrderNotProfitableError(...)
```

> 💡 **Tip:** Avoid duplicating rate constants across modules. Reference `GRADE_DISCOUNT_RATE` directly so future rate changes don't require hunting down hardcoded values.
