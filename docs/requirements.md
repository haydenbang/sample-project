# 쇼핑몰 백오피스 기능 요구사항

## 1. 회원 관리

### 1.1 회원 목록 조회
- 전체 회원 목록을 페이지네이션하여 조회할 수 있어야 한다.
- 상태(`active` / `inactive`)로 필터링할 수 있어야 한다.
- 이메일, 아이디로 검색할 수 있어야 한다.
- 가입일 기준 정렬을 지원해야 한다.

### 1.2 회원 상세 조회
- 회원 ID로 특정 회원의 상세 정보를 조회할 수 있어야 한다.
- 해당 회원의 주문 이력을 함께 조회할 수 있어야 한다.

### 1.3 회원 등록
- 아이디, 이메일, 비밀번호를 입력받아 신규 회원을 등록할 수 있어야 한다.
- 이메일, 아이디 중복 체크를 수행해야 한다.
- 비밀번호는 bcrypt로 해시하여 저장해야 한다.

### 1.4 회원 정보 수정
- 이메일, 계정 상태를 수정할 수 있어야 한다.
- 비밀번호 변경은 별도 API를 통해 처리해야 한다.

### 1.5 회원 비활성화
- 회원 계정을 비활성화(`inactive`) 처리할 수 있어야 한다.
- 비활성화된 회원은 로그인이 불가해야 한다.

## 2. 주문 관리

### 2.1 주문 목록 조회
- 전체 주문 목록을 페이지네이션하여 조회할 수 있어야 한다.
- 주문 상태(`pending` / `confirmed` / `shipped` / `cancelled`)로 필터링할 수 있어야 한다.
- 주문일 범위로 필터링할 수 있어야 한다.
- 회원 ID로 특정 회원의 주문만 조회할 수 있어야 한다.

### 2.2 주문 상세 조회
- 주문 ID로 특정 주문의 상세 정보를 조회할 수 있어야 한다.
- 주문 상품, 수량, 단가, 할인율, 최종 금액을 포함해야 한다.
- 배송 정보(수령인, 주소, 연락처, 배송비)를 함께 조회할 수 있어야 한다.

### 2.3 주문 생성
- 회원 ID, 상품 ID, 수량, 배송 정보를 입력받아 주문을 생성할 수 있어야 한다.
- 배송 정보는 수령인 이름, 배송 주소, 연락처, 배송비를 포함해야 한다.
- 상품 재고를 확인하고, 재고 부족 시 주문 생성을 거부해야 한다.
- 주문 시점의 상품 단가를 기록해야 한다.
- 수량 기반 할인율을 자동 계산하여 최종 금액을 산정해야 한다.
- 최종 금액은 상품 금액(할인 적용) + 배송비로 계산해야 한다.

### 2.4 배송 정보 관리
- 주문 생성 시 필수로 배송 정보를 입력해야 한다.
- 배송비는 기본 3,000원이며, 상품 또는 조건에 따라 변경될 수 있다.
- 배송 주소는 최대 500자까지 입력 가능하다.
- 수령인 이름과 연락처는 필수 입력 항목이다.

### 2.5 주문 상태 변경
- 주문 상태를 `pending` → `confirmed` → `shipped` 순서로 변경할 수 있어야 한다.
- 주문 취소(`cancelled`)는 `pending` 또는 `confirmed` 상태에서만 가능해야 한다.
- 주문 취소 시 재고가 원복되어야 한다.

### 2.6 주문 통계
- 일별, 월별 주문 수 및 매출 합계를 조회할 수 있어야 한다.

## 3. 상품 관리

### 3.1 상품 목록 조회
- 전체 상품 목록을 페이지네이션하여 조회할 수 있어야 한다.
- 카테고리, 판매 여부로 필터링할 수 있어야 한다.
- 상품명으로 검색할 수 있어야 한다.

### 3.2 상품 상세 조회
- 상품 ID로 특정 상품의 상세 정보를 조회할 수 있어야 한다.

### 3.3 상품 등록
- 상품명, 설명, 가격, 재고, 카테고리를 입력받아 상품을 등록할 수 있어야 한다.
- 가격은 0보다 큰 값이어야 한다.
- 재고는 0 이상의 정수여야 한다.

### 3.4 상품 정보 수정
- 상품명, 설명, 가격, 카테고리, 판매 여부를 수정할 수 있어야 한다.

### 3.5 재고 관리
- 재고 수량을 직접 조정할 수 있어야 한다.
- 재고가 0이 되면 자동으로 품절 처리(`is_active=False`)할 수 있어야 한다.

### 3.6 상품 삭제
- 상품을 비활성화(`is_active=False`) 처리할 수 있어야 한다.
- 이미 주문된 상품은 실제 삭제하지 않고 비활성화만 수행해야 한다.

## 4. 인증 및 권한

### 4.1 로그인
- 아이디/비밀번호로 로그인하고 JWT 토큰을 발급받아야 한다.
- 토큰 유효 시간은 24시간이다.

### 4.2 토큰 갱신
- 만료 전 토큰을 갱신할 수 있어야 한다.

### 4.3 권한 제어
- 모든 백오피스 API는 인증된 사용자만 접근 가능해야 한다.
