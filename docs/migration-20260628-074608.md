# Migration Guide: `final_amount` Field Added to `OrderOut`

**Date:** 2026-06-28
**Severity:** CRITICAL / HIGH
**Affected Teams:** Backend, Frontend, QA

---

## Summary

A new non-nullable integer field, `final_amount`, has been added to the `OrderOut` response schema. This field is required (`nullable: False`) at both the database and serialization layers, meaning every code path that creates, updates, seeds, or returns an `Order` object must now supply a valid integer value for `final_amount`. Without this field, all order-related API endpoints will fail with Pydantic `ValidationError` or database `NOT NULL` constraint violations, and the frontend will display `undefined` or crash outright. This change affects 7 files across the backend, frontend, and test layers and must be fully addressed before deployment.

---

## Breaking Changes

### 1. `OrderOut` Pydantic Schema — Missing Field Declaration

The `OrderOut` schema does not declare `final_amount`, causing a `ValidationError` whenever an `OrderOut` instance is constructed.

**Before (`backend/app/models/order.py`):**
```python
class OrderOut(BaseModel):
    id: int
    customer_id: int
    status: str
    created_at: datetime
```

**After:**
```python
class OrderOut(BaseModel):
    id: int
    customer_id: int
    status: str
    created_at: datetime
    final_amount: int  # NEW — required, non-nullable
```

---

### 2. Order Service — ORM Object Not Setting `final_amount`

The order service creates or updates `Order` ORM instances without setting `final_amount`, causing `NOT NULL` constraint violations at the database level.

**Before (`backend/app/services/order_service.py`):**
```python
def create_order(db: Session, data: OrderCreate) -> OrderOut:
    order = Order(
        customer_id=data.customer_id,
        status="pending",
    )
    db.add(order)
    db.commit()
    db.refresh(order)
    return OrderOut.from_orm(order)
```

**After:**
```python
def create_order(db: Session, data: OrderCreate) -> OrderOut:
    final_amount = compute_final_amount(data)  # implement your business logic
    order = Order(
        customer_id=data.customer_id,
        status="pending",
        final_amount=final_amount,  # NEW — must be set before commit
    )
    db.add(order)
    db.commit()
    db.refresh(order)
    return OrderOut.from_orm(order)
```

> ⚠️ **Note:** `compute_final_amount` is a placeholder. Replace it with your actual pricing/discount logic (e.g., sum of line items, applied discounts, tax).

### 3. API Endpoints — Serialization Failures on All Order Routes

All endpoints returning `OrderOut` will raise a `ValidationError` until the schema and ORM model are updated.

**Affected endpoints:**
| Method | Route |
|--------|-------|
| `GET` | `/orders` |
| `GET` | `/orders/{id}` |
| `POST` | `/orders` |
| `PATCH` | `/orders/{id}` |

**Before (`backend/app/routers/orders.py`):**
```python
@router.get("/orders/{order_id}", response_model=OrderOut)
def get_order(order_id: int, db: Session = Depends(get_db)):
    order = db.query(Order).filter(Order.id == order_id).first()
    if not order:
        raise HTTPException(status_code=404, detail="Order not found")
    return order  # ❌ Will raise ValidationError — final_amount missing
```

**After:**
```python
@router.get("/orders/{order_id}", response_model=OrderOut)
def get_order(order_id: int, db: Session = Depends(get_db)):
    order = db.query(Order).filter(Order.id == order_id).first()
    if not order:
        raise HTTPException(status_code=404, detail="Order not found")
    return order  # ✅ Works once OrderOut schema and ORM model are updated
```

> The router code itself may not need changes beyond ensuring the service layer and schema are correct. Verify each endpoint's response path populates `final_amount`.

### Alembic Migration (add column with backfill)

```python
# migrations/versions/xxxx_add_final_amount_to_orders.py

from alembic import op
import sqlalchemy as sa


def upgrade():
    # Step 1: Add column as nullable to allow backfill
    op.add_column("orders", sa.Column("final_amount", sa.Integer(), nullable=True))

    # Step 2: Backfill existing rows — replace 0 with your actual default/computed value
    op.execute("UPDATE orders SET final_amount = 0 WHERE final_amount IS NULL")

    # Step 3: Enforce NOT NULL constraint
    op.alter_column("orders", "final_amount", nullable=False)


def downgrade():
    op.drop_column("orders", "final_amount")
```
